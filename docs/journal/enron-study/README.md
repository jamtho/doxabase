# Enron Emails DoxaBase Knowledge Store

This directory contains a DoxaBase knowledge store built from the local MinIO
`enron-emails` bucket in a clean case-study run.

The store is meant to help future agents and analysts start from reviewed
dataset context rather than rediscovering schema, joins, caveats, and safe
analysis populations.

## Main Files

- `enron-emails.doxabase.sqlite` - the local DoxaBase capsule.
- `enron_knowledge_store.trig` - importable project TriG export.
- `enron_knowledge_store_handoff_manifest.json` - preferred handoff manifest.
- `enron_knowledge_store_handoff.trig` - handoff TriG artifact.
- `enron_knowledge_store_revision_snapshots.json` - revision snapshot bundle.
- `enron_doxabase_case_study_report.md` - detailed case-study report.
- `enron_docs_overview.md` - human narrative overview of the docs/artifacts.
- `enron_starter_tasks.md` - starter task checklist for future agents.
- `enron_code_owner_feedback.md` - DoxaBase product/API feedback from the trial.
- `enron_query_cookbook.md` - starter query patterns.
- `enron_analysis_views.md` - named logical analysis populations and view SQL.
- `western_power_policy_analysis.md` - deeper Western power policy subcorpus
  readout.
- `western_power_policy_lane_analysis.md` - four-lane Western power policy
  follow-up with recommended case studies.
- `verify_enron_handoff.py` - fresh-capsule handoff verification script.
- `enron_handoff_verify_latest_summary.json` - generated by the verification
  script when rerun.

## Quick Start

Open the capsule with the fresh DoxaBase checkout used for this run:

```bash
cd /tmp/doxabase-enron-clean.pClwOo/doxabase
PYTHONPATH=/tmp/doxabase-enron-clean.pClwOo/doxabase \
UV_CACHE_DIR=/tmp/doxabase-enron-clean.pClwOo/uv-cache \
uv run python - <<'PY'
from doxabase import DoxaBase, to_dict
db = DoxaBase('/tmp/doxabase-enron-clean.pClwOo/enron-emails.doxabase.sqlite')
print(to_dict(db.validate_graph(scope='all'))['conforms'])
print(to_dict(db.project_brief(limit=100, profile_candidate_limit=50))['key_counts'])
PY
```

Start investigation from these DoxaBase calls:

```python
db.project_brief(limit=100, profile_candidate_limit=50)
db.describe_dataset("https://example.test/enron-emails#eml_messages")
db.describe_dataset("https://example.test/enron-emails#xml_messages")
db.describe_context_slice(
    ["https://example.test/enron-emails#eml_messages"],
    profile="dataset_brief",
)
db.list_entities(type="rc:Pattern", graph="patterns", limit=50)
```

Verify the handoff import with an external checkout:

```bash
DOXABASE_CHECKOUT=/tmp/doxabase-enron-clean.pClwOo/doxabase \
uv run python verify_enron_handoff.py
```

## High-Level Guidance

Use `doc_id` for message identity and `parent_doc_id` for attachment joins.
Choose EML or XML deliberately: EML has broader message coverage and body text;
XML is a strict subset with richer native path/hash metadata. Name your analysis
population before reporting counts or models, because date filters, folder
families, body/body_top choice, sender parsing, and EML/XML representation all
change denominators materially.
