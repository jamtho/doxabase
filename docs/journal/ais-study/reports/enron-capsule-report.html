● ModerateRegex extraction finds useful domains but misses many populated sender/recipient strings. For example, EML to_addrs is populated on 652,110 rows but regex email tokens appear in only 215,775 rows; XML to is populated on 746,550 rows with regex tokens in 311,860 rows. Some fields contain Exchange/X.500-style markers.
Impact: Network analysis needs a parser strategy and coverage audit, not raw string grouping alone.
● ModerateRecommended logical populations include plausible-date EML messages (1,210,548 rows), plausible-date XML messages (1,203,416 rows), EML/XML overlap (1,227,255 rows), EML-only blank-source subset (7,132 rows), body-text rows (739,872), body_top-text rows (695,225), EML regex sender rows (97,929), and XML regex sender rows (198,050).
Impact: Analysts should name and preserve the population definition behind every count, model, network, or NLP run.
● ModerateBoth EML and XML attachment-row counts reach 260 attachments for one message. The p99 is 4 for EML and 5 for XML, p999 is 33 for both, and 4,525 messages have 10 or more attachments.
Impact: Attachment analyses should use robust summaries and inspect high-attachment messages separately.
● ModerateEML body_top equals body for 1,025,481 rows and is shorter than body for 208,906 rows. It is useful for top-level-message analysis but should be treated as derived from body.
Impact: Choose body or body_top deliberately for NLP and quote/context retrieval tasks.
● ModerateRegex domain extraction yields useful domain-level matrices, but many rows have no regex sender or recipient domain. EML has 1,001,806 rows in the none->none sender/recipient class bucket; XML has 895,661. XML has stronger regex sender coverage than EML because XML from_ often embeds Enron addresses.
Impact: Domain network analysis should report extractability and treat none-class rows as missing parser coverage, not absence of communication.
● ModerateMessage date fields contain outliers outside the usual Enron-era analysis window. EML has 23,463 rows before 1997, 115 rows after 2004, and 261 null dates; XML has 23,492 rows before 1997 and 347 rows after 2004, with extreme parsed years from 1316 to 9719.
Impact: Time-series analysis should apply an explicit date window or outlier policy.
● ModerateDuplicate EML message_id groups cross custodians in 7,490 groups and folders in 6,823 groups. Duplicate non-empty body groups cross custodians in 9,917 groups and folders in 121,335 groups. XML attachment native_hash duplicate groups cross custodians in 11,771 groups.
Impact: Deduplication should be corpus-wide and should preserve custodian/folder provenance instead of deduplicating inside isolated slices only.
● ModerateThe 7,132 EML message rows absent from XML all have blank folder and blank source_file. They span 31 custodians, have dates from 2000-03-29 through 2002-01-07, and include 17 messages flagged with one attachment each.
Impact: Treat EML-only rows as a special coverage subset before comparing EML and XML analyses.
● ModerateSeveral EML string columns are physically non-null but contain empty strings: to_addrs 582,277 rows, cc_addrs 1,062,068 rows, subject 144,092 rows, body 494,515 rows, body_top 539,162 rows, and folder 114,683 rows.
Impact: Normalize empty strings to missing values before completeness, recipient, subject, or body analyses.
● ModerateEML doc_id is unique and non-null, but message_id has 25,462 duplicate groups, 64,332 rows in duplicate groups, and up to 73 rows sharing one message_id.
Impact: Use doc_id as the row key; use message_id only for conversation or duplicate analysis.
● ModerateA practical message-like population excludes calendar, contacts, and meetings after date filtering. That gives 980,561 EML rows, 714,325 EML rows with body text, 670,649 EML rows with body_top text, and 973,356 XML rows.
Impact: NLP or network analyses over ordinary messages should cite whether these families were excluded.
● ModerateThe largest plausible-date month is 2002-11 with 134,027 EML rows and 134,027 XML rows, but the EML folder-family/month profile attributes 126,208 of those EML rows to calendar. Naive monthly message-volume plots will overstate ordinary communications in that month.
Impact: Monthly time-series analysis should stratify by folder/location family and consider message-like exclusions.
● ModerateXML messages are a strict subset of EML by doc_id in the observed files. All 1,227,255 XML doc_ids appear in EML, while 7,132 EML rows, all with blank folder, do not appear in XML.
Impact: Do not treat EML/XML row counts as interchangeable without choosing a representation.
● ModerateEML from_email is populated for only 98,103 of 1,234,387 rows, while XML from_ uses display-name style strings and is populated for 972,498 of 1,227,255 rows.
Impact: Sender analytics need representation-specific parsing and should not simply join EML from_email to XML from_.
● ModerateEML x_sdoc and x_zlid each have 114,682 blank values, creating one large duplicate group. They are useful source metadata where populated but not universal keys.
Impact: Use doc_id for identity and treat x_sdoc/x_zlid as optional source metadata.
● ModerateEML has 25,462 duplicate message_id groups, 121,584 duplicate non-empty subject_clean groups, 128,273 duplicate non-empty body groups, and 117,996 duplicate non-empty body_top groups. reply_depth ranges up to 56.
Impact: Conversation or deduplication work should combine doc_id, message_id, subject_clean, reply_depth, folders, and text hashes.
